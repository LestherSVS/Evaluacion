Tarea 3 - Uso de Git y GitHub

¿Qué es Git?

Git es un sistema de control de versiones distribuido que nos permite gestionar de manera eficiente los cambios en nuestros archivos, especialmente en proyectos de desarrollo de software. Es utilizado para realizar un seguimiento de los cambios, colaborar con otros y mantener un historial detallado de los archivos.

¿Qué es GitHub?

GitHub es una plataforma en línea para alojar proyectos que utilizan Git como sistema de control de versiones. Permite colaborar con otros desarrolladores, compartir código, realizar seguimientos de errores y mucho más.

Pasos para usar Git y GitHub:
  1. Instalar Git: Si aún no tienes Git en tu máquina, descárgalo desde aquí.
  2. Configurar Git: Configura tu nombre y correo electrónico con los comandos: 
    git config --global user.name "Tu Nombre"
    git config --global user.email "tu-email@dominio.com"
  3. Crear un repositorio en GitHub: Ve a GitHub y crea un nuevo repositorio.
  4. Clonar el repositorio a tu máquina local: Usa el comando: 
    git clone https://github.com/tu-usuario/tu-repositorio.git
  5. Hacer cambios y hacer commit: Realiza cambios en tu proyecto y usa los siguientes comandos: 
    git add .
    git commit -m "Descripción de tus cambios"
  6. Subir los cambios a GitHub: Usa el comando: 
    git push origin main
  7. Verificar los cambios en GitHub: Abre tu repositorio en GitHub para ver los archivos actualizados.
Comandos Básicos de Git:
  • git init: Inicializa un repositorio local.
  • git clone [URL]: Clona un repositorio remoto.
  • git status: Muestra el estado de los archivos modificados.
  • git add .: Agrega todos los archivos modificados al área de preparación.
  • git commit -m "mensaje": Realiza un commit con un mensaje explicativo.
  • git push origin main: Sube los cambios al repositorio remoto.
  • git pull origin main: Obtiene los cambios más recientes desde GitHub.
  • git log: Muestra el historial de commits.
